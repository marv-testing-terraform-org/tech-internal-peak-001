# Security Policy

## Supported Versions

We take security vulnerabilities seriously and are committed to supporting and addressing security issues in our project. Below is a list of versions that are currently supported with security updates:

| Version    | Supported          |
| ---------- | ------------------ |
| 1.x        | :white_check_mark:  |
| 0.x        | :x:                |

## Reporting a Vulnerability

If you discover a security vulnerability within this project, please follow the steps below:

1. **Do not open a public issue.** To protect the project's users, avoid publicly disclosing the vulnerability until we have assessed and addressed it.
   
2. Raise a Service Desk ticket <insert service desk url here>  with the following information:
   - A detailed description of the vulnerability.
   - Steps to reproduce the vulnerability.
   - Any relevant information such as the project version, environment details, and potential impact.

3. Our security team will review the issue and respond as quickly as possible. We will:
   - Acknowledge the receipt of your report within 48 hours.
   - Provide feedback and a timeline for addressing the vulnerability.
   - Keep you updated on our progress.

4. We ask that you give us a minimum of 90 days to fix the issue before making any public disclosures.

## Preferred Languages

Please submit vulnerability reports in **English**.

## Responsible Disclosure

We greatly appreciate security researchers who work with us responsibly to disclose vulnerabilities. If you follow the steps above, we commit to publicly acknowledging your contributions once the issue is resolved.
